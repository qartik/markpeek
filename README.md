# markpeek

Small local Markdown editor with a live preview.

## Run

```bash
npm install
npm run dev
```

Open the Vite URL printed in the terminal.

## Deploy

This app is fully static. A production build writes HTML, CSS, and JavaScript to
`dist/`.

```bash
npm run build
```

The repository includes a GitHub Pages workflow for publishing to:

```text
https://qartik.github.io/markpeek/
```

## What is reused or mirrored from Discourse

- Rendering uses `markdown-it`, the same base Markdown engine used by `frontend/discourse-markdown-it`.
- Tables are wrapped in `div.md-table`, matching Discourse's cooked table shape so wide tables can scroll.
- Fenced code blocks use `lang-*` classes, following the shape generated by Discourse's Markdown code feature.
- The toolbar intentionally mimics composer-style Markdown insertion from `frontend/discourse/app/lib/composer/toolbar.js`: bold, italic, link, quote, code, list, and heading actions use the same Markdown markers and common keyboard shortcuts where practical.
- Text manipulation mirrors the textarea-first decisions from `frontend/discourse/app/lib/textarea-text-manipulation.js`: selected text is trimmed before inline surrounds, surrounds can toggle off, list-like actions operate on line prefixes, focus is restored without changing scroll position, and final selection is preserved after toolbar actions.
- Paste handling mirrors the standalone-friendly parts of Discourse rich paste: HTML is converted to Markdown, tab-delimited spreadsheet data becomes Markdown tables, and pasting a URL over selected text creates a Markdown link.
- Undo and redo are implemented locally for both typing and toolbar changes so programmatic Markdown insertions can be reverted with toolbar buttons or keyboard shortcuts.
- Editor and preview scroll positions are synchronized by scroll ratio, matching the composer-preview expectation that both panes stay aligned while editing longer posts.
- Preview HTML is sanitized with `DOMPurify`, matching the same security boundary Discourse maintains after Markdown is cooked.

## Acknowledgements

markpeek is inspired by the [Discourse](https://github.com/discourse/discourse)
composer/editor experience. It does not include the Discourse runtime or forum
features, but it intentionally mirrors some Discourse editor decisions:

- Rendering follows `discourse-markdown-it` patterns where practical, using
  standalone `markdown-it`.
- Text-editing behavior is adapted from the textarea-first Discourse composer
  model: surrounds, line-prefix formatting, selection preservation, and
  scroll-preserving focus.
- Preview markup keeps Discourse-like conventions such as `div.md-table` table
  wrapping and `lang-*` code classes.

## What is approximated

- The full `frontend/discourse-markdown-it` package depends on Discourse workspace packages, Ember-facing aliases, site settings, upload helpers, mentions, oneboxes, and cooked-post decorators. This app uses a standalone `markdown-it` setup instead of importing that package directly.
- Syntax highlighting uses `highlight.js` locally. Real Discourse highlighting can vary by site configuration and server-side cooked output.
- Line breaks use `markdown-it`'s `breaks` option to feel close to composer preview behavior.

## Intentionally omitted

- Uploads and upload markdown resolution.
- Mentions, hashtag lookup, emoji autocomplete, oneboxes, watched words, censored words, drafts, auth, forum APIs, and server-rendered cooked post decorators.
- Rich editor mode and ProseMirror behavior.

## Files

- `src/markdown.ts` configures Markdown rendering and sanitization.
- `src/paste.ts` converts rich paste content to Markdown.
- `src/text-manipulation.ts` preserves selection and cursor behavior for formatting actions.
- `src/history.ts` manages local undo and redo snapshots.
- `src/scroll-sync.ts` links editor and preview scroll positions.
- `src/storage.ts` persists the draft to `localStorage`.
- `src/styles.css` provides the split-pane editor and cooked-preview styling.
