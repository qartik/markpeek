# markpeek

markpeek is a fast, browser-only, Discourse-inspired Markdown editor for
drafting posts, notes, and docs with a live rendered preview.

Open it here: [https://qartik.github.io/markpeek/](https://qartik.github.io/markpeek/)

## Why use it

- Write Markdown in a focused split-pane editor.
- See sanitized preview output as you type.
- Paste rich HTML and spreadsheet data into usable Markdown.
- Use familiar composer-style formatting buttons and keyboard shortcuts.
- Keep drafts local in your browser with no backend, accounts, or networked
  document storage.

## Smart Paste

markpeek handles common paste workflows directly in the browser:

- HTML paste becomes Markdown.
- Tab-delimited spreadsheet text becomes a Markdown table.
- Pasting a URL over selected text creates a Markdown link.

It intentionally does not implement uploads, oneboxes, mentions, auth, drafts
sync, forum APIs, or any Discourse server behavior.

## Composer-Style Editing

The toolbar covers the Markdown actions used most while drafting:

- Bold, italic, link, quote, inline/code block, bullet list, numbered list, and
  heading cycling.
- Undo and redo for both typing and toolbar actions.
- Scroll sync between the editor and preview for longer drafts.

Keyboard shortcuts include:

| Action | Shortcut |
| --- | --- |
| Bold | `Mod+B` |
| Italic | `Mod+I` |
| Link | `Mod+K` |
| Code | `Mod+E` |
| Quote | `Mod+Shift+9` |
| Heading cycle | `Shift+Mod+H` |
| Undo | `Mod+Z` |
| Redo | `Shift+Mod+Z` or `Mod+Y` |

`Mod` means Command on macOS and Control on Windows/Linux.

## Discourse Attribution

markpeek is inspired by the [Discourse](https://github.com/discourse/discourse)
composer/editor experience. It does not include the Discourse runtime or forum
features, but it intentionally mirrors standalone-friendly editor behavior:

- Rendering uses `markdown-it`, the same base Markdown engine used by
  `frontend/discourse-markdown-it`.
- Tables are wrapped in `div.md-table`, matching Discourse's cooked table shape
  so wide tables can scroll.
- Fenced code blocks use `lang-*` classes, following the shape generated by
  Discourse's Markdown code feature.
- Toolbar behavior follows composer-style Markdown insertion from
  `frontend/discourse/app/lib/composer/toolbar.js`.
- Text manipulation mirrors textarea-first decisions from
  `frontend/discourse/app/lib/textarea-text-manipulation.js`.
- Paste handling mirrors the standalone-friendly parts of Discourse rich paste.
- Preview HTML is sanitized with `DOMPurify`, matching the same security
  boundary Discourse maintains after Markdown is cooked.

See [NOTICE.md](NOTICE.md) for the attribution summary.

## What Is Approximated

- The full `frontend/discourse-markdown-it` package depends on Discourse
  workspace packages, Ember-facing aliases, site settings, upload helpers,
  mentions, oneboxes, and cooked-post decorators. markpeek uses a standalone
  `markdown-it` setup instead.
- Syntax highlighting uses `highlight.js` locally. Real Discourse highlighting
  can vary by site configuration and server-side cooked output.
- Line breaks use `markdown-it`'s `breaks` option to feel close to composer
  preview behavior.

## Project Map

- `src/markdown.ts` configures Markdown rendering and sanitization.
- `src/paste.ts` converts rich paste content to Markdown.
- `src/text-manipulation.ts` preserves selection and cursor behavior for
  formatting actions.
- `src/history.ts` manages local undo and redo snapshots.
- `src/scroll-sync.ts` links editor and preview scroll positions.
- `src/storage.ts` persists the draft to `localStorage`.
- `src/styles.css` provides the split-pane editor and cooked-preview styling.
